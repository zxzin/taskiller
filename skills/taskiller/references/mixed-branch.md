# Mixed Branch

Use this branch when the project includes multiple artifact types and no single input type fully describes the work.

## Default Rule

Choose the primary route by final handoff, not by number of files.

Examples:

- report plus spreadsheet appendix -> primary `document`, secondary `data`
- pitch deck plus working demo -> primary `presentation`, secondary `code`
- software change plus summary memo -> primary `code`, secondary `document`

## First Working Version

The first working version should establish the primary deliverable first.

Secondary artifacts should support the primary handoff, not become parallel full projects unless the requirements explicitly demand that.

## Execution Priorities

1. identify the real final handoff artifact
2. choose the primary route
3. build the primary route's first working version
4. add only the supporting secondary outputs needed for completeness
5. validate the full package as one delivery

## Validation Focus

- the primary deliverable is complete enough to stand on its own
- secondary outputs are present only when they materially support the handoff
- the project does not sprawl into unnecessary artifact generation

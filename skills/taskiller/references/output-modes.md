# Output Modes

Use these structures as defaults for the outward-facing deliverable. Choose the smallest structure that still produces a submission-ready result.

## 1. Project Brief

Use for early-stage consolidation when the user needs one clean project view.

```text
Project Objective
Background
Key Requirements
Recommended Output
Pending Confirmations
Next Steps
```

## 2. Requirements Document

Use when the final output must guide implementation or review.

```text
Objective
Business or Project Context
Functional Requirements
Non-Functional Requirements
Scope Boundaries
Dependencies and Risks
Acceptance Criteria
```

## 3. Execution Plan

Use when the user already knows what to do but needs a clean delivery path.

```text
Target Result
Current Inputs
Execution Phases
Owner or Role Split
Risks and Blockers
Milestones or Submission Path
```

## 4. Submission Draft

Use when the output is going directly to another person or group.

```text
Purpose
Final Statement or Proposal
Supporting Points
Requested Decision or Next Action
```

## Selection Rule

- If the recipient needs to approve, prefer `Project Brief` or `Submission Draft`.
- If the recipient needs to build, prefer `Requirements Document`.
- If the recipient needs to execute internally, prefer `Execution Plan`.

If uncertain, start from `Project Brief` and tighten it.

## Reminder

These are external output modes. They do not replace the internal project control files defined in `project-state-files.md`.
